## 0. The problem to fix

The Phase 1 Lean substrate (`lean/ZetaRH/LambdaBlueprints.lean`) writes the Fermat-Frobenius axiom as

```
psi_fermat_frobenius : ∀ p : Nat.Primes, ∀ x : carrier,
  (psi p x, x ^ (p.val)) ∈ relations
```

This is too coarse. With `relations := Set.univ` (as in the Phase 1 `F1` and `integers_as_lambda_blueprint` instances) the axiom is satisfied vacuously by any `psi`. In particular, `psi := id` on `Z` satisfies the axiom not because of Fermat's little theorem but because `Set.univ` contains everything. We need a refinement where the membership statement is non-vacuous: the relation set must contain enough information to *distinguish* honest mod-p congruences from arbitrary pairs.

The Joyal/Borger condition on Lambda-rings is `psi_p(x) - x^p ∈ p R`. Our task is to lift this to blueprints, where addition is only a partial multivalued operation and "p R" is not literally available as an ideal.

## 1. Definition: the p-congruence sub-relation `congr_p`

Let `(B, B^bullet)` be a Lorscheid blueprint, with `B` a commutative semiring, `B^bullet ⊆ B` a multiplicative submonoid containing `1`, and `relations ⊆ B × B` a set of forced equalities (the "pre-additive" relations that pass to `≈`, the congruence on `N[B^bullet]` generated by them).

For each rational prime `p` (encoded as the integer `p ∈ N ⊆ B` via the canonical semiring map `N → B`, `n ↦ n · 1_B`), we define a sub-relation

```
congr_p ⊆ B × B
```

as the smallest sub-relation of `relations` closed under the four rules below, containing the canonical generator set `G_p`.

### 1.1 The canonical generator set `G_p`

`G_p ⊆ B × B` is the union of three families:

(G1) **Multiples-of-p generators**: for every `m ∈ B`, the pair `(p · m, 0_B) ∈ G_p`. Informally: `p · m ≡ 0` modulo `p`. (Here `p · m` is shorthand for `(p · 1_B) * m`, which is computed in the semiring `B`. The element `p · 1_B` always exists in `B` because `N → B` is the canonical structure map.)

(G2) **Reflexivity restricted to `B^bullet`**: for every `x ∈ B^bullet`, the pair `(x, x) ∈ G_p`. (This is included so that elements of the pointed monoid are always congruent to themselves; the rest of `B` will inherit reflexivity through the closure rule R0 below.)

(G3) **Frobenius generators on `B^bullet`** (added if and only if the candidate is the "free Fermat-Frobenius candidate"; optional otherwise): the empty set when defining `congr_p` from data, OR `{ (psi_p(x), x^p) | x ∈ B^bullet }` when we want to embed the F-F axiom into the generators rather than impose it externally. In this document we keep (G3) empty in the definition of `congr_p` and impose F-F as a separate axiom in §2.

### 1.2 The four closure rules

`congr_p` is the smallest subset of `B × B` containing `G_p` and closed under:

(R0) **Reflexivity, symmetry, transitivity**: `(x, x) ∈ congr_p`; if `(a, b) ∈ congr_p` then `(b, a) ∈ congr_p`; if `(a, b), (b, c) ∈ congr_p` then `(a, c) ∈ congr_p`.

(R1) **Additivity within the blueprint**: if `(a, b), (a', b') ∈ congr_p` AND the pairs `(a + a', a' + a)` and `(b + b', b' + b)` are all in `relations` (i.e., the additions are blueprint-legal), then `(a + a', b + b') ∈ congr_p`. The condition "the additions are blueprint-legal" is shorthand for "the relation set `relations` already records the addition as a valid operation"; for honest rings this is automatic because every addition is legal.

(R2) **Multiplicativity by `B^bullet`**: if `(a, b) ∈ congr_p` and `m ∈ B^bullet`, then `(m * a, m * b) ∈ congr_p`. (Multiplication by elements outside `B^bullet` is not allowed; this is the blueprint-style refinement of "multiplication is compatible with congruence".)

(R3) **Containment in `relations`**: `congr_p ⊆ relations`. Equivalently: `relations` must already contain all of `congr_p`. This is a *consistency demand* on the blueprint, not a closure rule per se. A blueprint that does not satisfy `congr_p ⊆ relations` is said to be *incompatible with mod-p arithmetic* and cannot be promoted to a Lambda-blueprint at the prime `p`.

### 1.3 Intuition

`congr_p` is the blueprint-level version of "the congruence relation `a ≡ b (mod p)`". The generators (G1) say "anything of the form `p · m` is zero mod p"; the closure rules (R0)-(R2) say "this is a congruence relation on `B` for the subset of additions and the multiplications that are blueprint-legal"; (R3) says "the blueprint's own relations must record all these mod-p equalities". A blueprint that ignores mod-p arithmetic (no `(p · m, 0)` pair in `relations` for any `m ≠ 0`) cannot host a Lambda-structure at `p`.

### 1.4 Comparison with §2.2 of the parent dossier

R2.5 §2.2 defined `B/p` as "the blueprint where `B` is unchanged but the relations are extended by `p · 1_B = 0_B`". The current Definition 1.1 makes that precise:

- "Extended by `p · 1_B = 0`" in R2.5 corresponds to generator family (G1) above.
- "B/p has a blueprint structure" is the assertion that the closure under (R0)-(R2) produces a blueprint congruence. Lemma 1.5 below formalizes this.
- The R2.5 diagram-commutativity formulation is recovered as the special case `(psi_p(x), x^p) ∈ congr_p` (Definition 2.1 below).

The difference from R2.5: we DO NOT extend the entire blueprint; we identify a sub-relation of `relations`. This keeps `B` itself unchanged and makes the Lean encoding straightforward (`congr_p` is a `Set (carrier × carrier)`, not a separate blueprint).

### 1.5 Lemma (well-definedness)

For every blueprint `(B, B^bullet, relations)` satisfying (R3), the sub-relation `congr_p` is uniquely defined and is itself an equivalence relation on `B` compatible with multiplication by `B^bullet` and with the blueprint-legal additions.

**Proof sketch.** Smallness is by intersection: `congr_p` is the intersection of all subsets of `relations` containing `G_p` and closed under (R0)-(R2). The intersection is nonempty because `relations` itself is one such set when (R3) holds. The closure rules (R0)-(R2) are preserved by arbitrary intersection (each rule is of the form "if some pairs are in the set, then some other pair is in the set", which is preserved by `cap`), so the intersection itself is closed. Uniqueness is from smallness. Equivalence-relation properties are R0. Compatibility with multiplication by `B^bullet` is R2. Compatibility with blueprint-legal additions is R1.

## 2. Definition: the Fermat-Frobenius axiom in blueprint-relation form

A Lambda-blueprint `(B, B^bullet, relations, {psi_p})` satisfies the **blueprint-relation Fermat-Frobenius axiom** if:

(FF-A) For every prime `p` and every `x ∈ B`, the pair `(psi_p(x), x^p) ∈ congr_p`.

Notice that we require membership in `congr_p`, not in `relations` directly. This is the precise sense in which `psi_p(x) - x^p` is "divisible by p in the blueprint".

For a typical blueprint, `congr_p` is much smaller than `relations` (since `congr_p` is generated only from the mod-p generators (G1), whereas `relations` records all forced equalities). So (FF-A) is a non-vacuous demand: it says that the specific pair `(psi_p(x), x^p)` lies in the sub-relation generated by mod-p arithmetic, not just any old forced equality.

### 2.1 Why this fixes the Phase 1 substrate

The Phase 1 axiom `(psi p x, x ^ p.val) ∈ relations` is satisfied trivially when `relations := Set.univ` (the case used in the placeholder `F1` and `integers_as_lambda_blueprint`).

The refined axiom (FF-A) is NOT satisfied trivially even when `relations := Set.univ`, because `congr_p` is the *smallest* sub-relation closed under (R0)-(R2) containing `G_p`. With `relations := Set.univ`, the closure rule (R3) `congr_p ⊆ relations` is trivially satisfied, but `congr_p` itself is computed from `G_p` and is small.

For example, with `B = Z`, `B^bullet = Z` (the entire ring is its own pointed monoid since `Z` is a ring), `relations = Set.univ`:

- `G_p` = `{ (p m, 0) | m ∈ Z } ∪ { (x, x) | x ∈ Z }`.
- Closing under (R0)-(R2) gives `congr_p = { (a, b) ∈ Z × Z | a ≡ b (mod p) }`. This is the honest mod-p congruence on `Z`.
- (FF-A) with `psi_p = id` then asks `(x, x^p) ∈ congr_p`, i.e., `x ≡ x^p (mod p)`. This is exactly Fermat's little theorem.

So even though `relations := Set.univ` (which makes the OLD axiom trivial), the NEW axiom (FF-A) imposes Fermat's little theorem on `Z`. The Phase 1 substrate is not falsified, it is sharpened.

## 3. Specialization Lemma 1 (recovery of Joyal/Borger)

**Statement.** Let `(B, B^bullet, relations)` be a blueprint that is *a ring*, in the sense that:

- `B^bullet = B` (every element is pointed), and
- `relations` records every additive equality (i.e., `(a + b, b + a) ∈ relations` for all `a, b`, and `relations` is the full congruence generated by the ring axioms).

Then `congr_p` (as defined in §1) is exactly the principal ideal congruence modulo `p B`: that is, `(a, b) ∈ congr_p` iff `a - b ∈ p B`.

Consequently, the blueprint-relation Fermat-Frobenius axiom (FF-A) for `(B, B^bullet, relations, {psi_p})` is equivalent to the Joyal/Borger condition `psi_p(x) - x^p ∈ p B` for all `x ∈ B`.

**Proof.**
- **(forward)** Show `congr_p ⊆ {(a, b) | a - b ∈ p B}`. By induction on the closure construction. The generators (G1) `(p m, 0)` give `p m - 0 = p m ∈ p B`. (G2) `(x, x)` give `x - x = 0 ∈ p B`. The rules R0 (equivalence) and R1 (additivity, which is unrestricted since `B` is a ring) and R2 (multiplication by `B^bullet = B`) all preserve the property "`a - b ∈ p B`": R0 is symmetry/transitivity of equivalence which holds for `p B` as an ideal congruence; R1 says `(a + a') - (b + b') = (a - b) + (a' - b') ∈ p B` if both summands are; R2 says `m a - m b = m (a - b) ∈ p B` if `a - b ∈ p B`. So `congr_p` is contained in the principal ideal congruence.
- **(reverse)** Show `{(a, b) | a - b ∈ p B} ⊆ congr_p`. Take `(a, b)` with `a - b = p m` for some `m ∈ B`. Then `(p m, 0) ∈ G_p` by (G1). Apply R1 (the blueprint is a full ring so all additions are legal) to `(p m, 0)` and `(b, b)` (which is in `G_p` by (G2)): we get `(p m + b, 0 + b) = (a, b) ∈ congr_p`.

Combining, `congr_p` equals `{(a, b) | a - b ∈ p B}`, the standard mod-p ideal congruence. The FF axiom then reads `(psi_p(x), x^p) ∈ congr_p` iff `psi_p(x) - x^p ∈ p B`, which is the Joyal/Borger condition. QED.

**Remark on cleanness.** The proof uses no special features of `B` other than its being a ring with `B^bullet = B` and full additive relations. So Lemma 1 holds for `Z`, `Z[T]`, `F_p`, `F_p[T_1, ..., T_n]`, and any commutative ring viewed as a blueprint in the canonical way.

## 4. Specialization Lemma 2 (recovery of identity / Tits behavior)

**Statement.** Let `(B, B^bullet, relations, {psi_p})` be a Lambda-blueprint with `psi_p = id_B` for all primes `p`. Then (FF-A) reduces to the condition

(*) For every prime `p` and every `x ∈ B`, the pair `(x, x^p) ∈ congr_p`.

Sub-cases:

(a) **`F_1`** (carrier = `{0, 1}` multiplicatively; `B^bullet = {0, 1}`; `relations = {(0, 0), (1, 1)}`, the trivial reflexive set): (*) holds because `1^p = 1` (so `(1, 1) ∈ congr_p` by (G2)) and `0^p = 0` (so `(0, 0) ∈ congr_p` by R0 reflexivity, applied to `0` which is in `B` though not necessarily in `B^bullet`). So `F_1` admits the F-F axiom with `psi_p = id` trivially.

(b) **`Z`** (carrier = `Z`; `B^bullet = Z`; `relations = Set.univ`): (*) is exactly `x ≡ x^p (mod p)`, Fermat's little theorem. Holds for all `x ∈ Z`. So `Z` with `psi_p = id` is a Lambda-blueprint by Fermat's little theorem.

(c) **`Z[T]` with the canonical ring structure** (carrier = `Z[T]`; `B^bullet = Z[T]`; `relations = Set.univ`): (*) is `T^p ≡ T (mod p)` in `Z[T]`. This is **FALSE**: `T^p - T` is not divisible by `p` in `Z[T]` (the coefficient of `T^p` in `T^p - T` is `1`, not a multiple of `p`). So `Z[T]` with `psi_p = id` is NOT a Lambda-blueprint. (The fix is to take `psi_p(T) = T^p`, recovering the standard Lambda-ring structure on `Z[T]` as the "free Lambda-ring on one generator".)

(d) **`F_p`** (carrier = `F_p`; `B^bullet = F_p`; `relations = Set.univ`): (*) is `x ≡ x^p (mod p)` in `F_p`, which is the small Fermat-Frobenius fixed-point identity `x = x^p` (since `F_p` is already mod `p`). Holds. So `F_p` with `psi_p = id` is a Lambda-blueprint. (Note: here `psi_p = id` agrees with the Frobenius on `F_p` because `Frob = x ↦ x^p` IS the identity on `F_p`.)

(e) **Pure Tits / Deitmar object** (carrier is a multiplicative monoid `M` with `relations = ∅` aside from reflexivity, viewed as a semiring via `N[M]/(relations)`): the F-F axiom asks `(x, x^p) ∈ congr_p`. The generators (G1) are `(p · m, 0)`, which are nontrivial in `N[M]`, and the closure produces `congr_p`. (*) holds iff `x` and `x^p` differ by a multiple of `p` in `N[M]/(relations)`. For Deitmar-style monoid schemes this typically fails for non-torsion elements, which is the structural reason why Deitmar's framework does not admit a Lambda-structure at any prime. This matches (and refines) the §3 R2.5 prediction that "F_1 trivially, Z by FLT, but generic blueprints do not have psi_p = id satisfying FF".

**Proof of Lemma 2.** Substitute `psi_p = id` in (FF-A); the LHS `psi_p(x) = x` and the RHS `x^p` give `(x, x^p) ∈ congr_p`, which is (*). The sub-cases follow by direct computation of `congr_p` in each example, as sketched above. QED.

### 4.1 Honest assessment of `Z[T]` and the "polynomial blueprint" example

The R2.5 dossier §3.5 was vague: it asserted that `Z[T]` with `psi_p(T) = T^p` works "modulo blueprint subtleties". Lemma 2(c) clarifies what those subtleties are:

- With `psi_p = id`: the F-F axiom fails (because `T^p ≠ T (mod p)` in `Z[T]`).
- With `psi_p(T) = T^p` extended multiplicatively/additively to all of `Z[T]`: the F-F axiom becomes `(T^p, T^p) ∈ congr_p`, trivially true by reflexivity. More generally, for a polynomial `f(T)`, `psi_p(f(T)) = f(T^p)`, and we need `(f(T^p), f(T)^p) ∈ congr_p`. This is the standard Adams-operation identity `psi_p(f(T)) ≡ f(T)^p (mod p)`, which holds by the freshman's dream applied to each monomial.

So the "free Lambda-blueprint on one generator" is `(Z[T], psi_p(T) = T^p)`, NOT `(Z[T], psi_p = id)`. This is consistent with Borger's free Lambda-ring construction.

## 5. Worked examples (computing `congr_p` explicitly)

For each example we compute `congr_p` explicitly, then check (FF-A) for the candidate `psi_p`.

### 5.1 `F_1` (the trivial blueprint)

- Carrier `B = {0, 1}` with `0 + 0 = 0`, `1 + 1 = 0` (`F_1` convention), `0 * a = 0`, `1 * a = a`.
- `B^bullet = {0, 1}`.
- `relations = { (0, 0), (1, 1), (0, 1+1), (1+1, 0) }` (the "1 + 1 = 0" relation makes `F_1` characteristic-2-like, OR `relations = { (0,0), (1,1) }` for the purely-multiplicative Deitmar convention; both work).

`G_p` for any prime `p`:
- (G1) `(p · m, 0)` for `m ∈ {0, 1}`. With `p · 1 = 0` in `F_1` (under the characteristic-2 convention or under any convention where `N → F_1` is `n ↦ 0` for `n ≥ 2` and `1 → 1`), this gives `(0, 0)`. Trivial.
- (G2) `(0, 0), (1, 1)`.

`congr_p = { (0, 0), (1, 1) }`.

(FF-A) check with `psi_p = id`: `(id(0), 0^p) = (0, 0) ∈ congr_p`. `(id(1), 1^p) = (1, 1) ∈ congr_p`. PASS.

So `F_1` is a Lambda-blueprint in the refined sense.

### 5.2 `Z` with `psi_p = id`

- `B = Z`, `B^bullet = Z`, `relations = Set.univ`.
- `G_p = { (p m, 0) | m ∈ Z } ∪ { (x, x) | x ∈ Z }`.
- Closure under (R0)-(R2): `congr_p = { (a, b) ∈ Z × Z | a ≡ b (mod p) }` (the standard mod-p ideal congruence, by Lemma 1 applied to `B = Z`).

(FF-A) check with `psi_p = id`: `(id(x), x^p) = (x, x^p)`. By Fermat's little theorem, `x ≡ x^p (mod p)`, so `(x, x^p) ∈ congr_p`. PASS.

So `Z` with `psi_p = id` is a Lambda-blueprint by Fermat's little theorem.

### 5.3 `F_p` with `psi_p = Frobenius` (which equals `id` on `F_p`)

- `B = F_p`, `B^bullet = F_p`, `relations = Set.univ`.
- `G_p = { (0, 0) } ∪ { (x, x) | x ∈ F_p }` (since `p · m = 0` in `F_p` for any `m`).
- `congr_p = { (a, b) ∈ F_p × F_p | a ≡ b (mod p) } = { (a, b) | a = b }` (the diagonal, since everything is already mod `p`).

(FF-A) check with `psi_p(x) = x^p = x` (Fermat little theorem in `F_p`): `(x^p, x^p) = (x, x)` (on the diagonal). PASS.

So `F_p` with `psi_p = Frobenius` is a Lambda-blueprint.

### 5.4 `F_p[T]` with `psi_p(T) = T^p` (free Lambda-blueprint on one generator over `F_p`)

- `B = F_p[T]` (the polynomial ring over `F_p` in one variable).
- `B^bullet = F_p[T]` (we treat it as a ring).
- `relations = Set.univ`.

`congr_p`:
- `G_p = { (p · f, 0) | f ∈ F_p[T] } ∪ { (g, g) | g ∈ F_p[T] }`. Since `p · f = 0` in `F_p[T]` (the characteristic is `p`), generator (G1) collapses to `{ (0, 0) }`.
- Closure: `congr_p = { (g_1, g_2) | g_1 = g_2 in F_p[T] }`. (Because no non-trivial mod-`p` relations exist; `F_p[T]` is already mod `p`.)

So `congr_p` is the diagonal `{ (g, g) }`.

(FF-A) check with `psi_p(T) = T^p`, extended multiplicatively/additively (so `psi_p(f(T)) = f(T^p)` for `f ∈ F_p[T]`):

We need `(psi_p(f(T)), f(T)^p) = (f(T^p), f(T)^p) ∈ congr_p`, i.e., `f(T^p) = f(T)^p` in `F_p[T]`.

By the **Frobenius identity (Freshman's Dream)** in characteristic `p`: for `f(T) = sum_i a_i T^i`, we have `f(T)^p = sum_i a_i^p T^{ip} = sum_i a_i T^{ip}` (since `a_i ∈ F_p` gives `a_i^p = a_i`) `= f(T^p)`. PASS.

So `F_p[T]` with `psi_p(T) = T^p` is a Lambda-blueprint over `F_p`.

### 5.5 `Z[T]` with `psi_p(T) = T^p` (free Lambda-blueprint on one generator over `Z`)

- `B = Z[T]`, `B^bullet = Z[T]`, `relations = Set.univ`.
- `G_p = { (p · f, 0) | f ∈ Z[T] } ∪ { (g, g) }`. Here `p · f ≠ 0` because the characteristic is `0`.
- `congr_p = { (g_1, g_2) | g_1 ≡ g_2 (mod p Z[T]) }` (the ideal-quotient congruence, by Lemma 1).

(FF-A) check with `psi_p(T) = T^p` extended as a ring endomorphism (`psi_p(f) = f(T^p)`):

We need `(f(T^p), f(T)^p) ∈ congr_p`, i.e., `f(T^p) ≡ f(T)^p (mod p Z[T])`.

By the **Freshman's Dream over `Z` modulo `p`**: in `Z[T]/(p Z[T]) = F_p[T]`, we have `f(T)^p = f(T^p)` (by §5.4 computation). Lifting back to `Z[T]`, this says `f(T)^p - f(T^p) ∈ p Z[T]`. PASS.

So `Z[T]` with `psi_p(T) = T^p` is a Lambda-blueprint. This is the canonical "free" object.

### 5.6 Non-example: `Z[T]` with `psi_p = id`

By Lemma 2(c), this fails: `(T, T^p) ∈ congr_p` would require `T - T^p ∈ p Z[T]`, but the leading coefficient of `T - T^p` is `-1`, not divisible by `p`. **FAIL.** This is the correct behavior: `psi_p = id` is not the right Lambda-structure on `Z[T]`; `psi_p(T) = T^p` is.

### 5.7 Non-example (Tits-style monoid blueprint): a monoid with no compatible `psi_p`

Take `M = N` (the natural numbers as a multiplicative monoid), realized as a blueprint with `B^bullet = N`, `B = N[N]/(simple relations)`. Try `psi_p = id`. Then `congr_p` is generated by `{(p m, 0) | m ∈ N[N]}`. Whether `(n, n^p) ∈ congr_p` depends on the relation set. For the "free" choice with `relations` only the obvious reflexive/symmetric/transitive closure, `n ≠ n^p` for `n ≥ 2` and `p ≥ 2`, and the elements `n - n^p` (in the semiring sense) are not blueprint-legal multiples of `p`. **FAIL.**

This is the correct structural reason why a Deitmar-style monoid scheme cannot host a non-trivial Lambda-structure without enriching the relations: the multiplicative monoid alone does not record enough mod-p arithmetic.

## 6. K2 (D-H) self-check

**Question.** Does the Davenport-Heilbronn L-function fit as a Lambda-blueprint construction in the blueprint-relation form?

**Answer.** No, for the same structural reason articulated in `2A_R1_DH_exclusion.md` §3.2 and §3.3 for the parent frameworks. The D-H function

D(s) = sin(alpha) L(s, chi_a) + cos(alpha) L(s, chi_b)

is a **linear combination** of two Dirichlet L-functions, where the coefficients sin(alpha) and cos(alpha) are transcendental real numbers determined by Davenport-Heilbronn's specific construction. To construct D-H as a Lambda-blueprint, one would need:

1. A blueprint `(B, B^bullet)` whose "zeta function" (defined via Adams operations and the trace) equals `D(s)`.
2. A family `{psi_p}` of Adams operations satisfying (FF-A).

**Structural obstruction at (1).** The Lambda-blueprint zeta is constructed as a product over primes of local factors `(1 - psi_p t)^{-1}` (or similar Euler-product structure inherited from the Adams operations: see Borger 2009 §10). This is multiplicative by construction. The D-H function has **no Euler product**: it is a linear combination of two functions, each of which has its own Euler product, but the SUM does not decompose into a single Euler product.

To force D-H to be a Lambda-blueprint zeta, one would need the local factor at each prime to be `(1 - psi_p t)^{-1}` for some `psi_p` whose trace gave `sin(alpha) chi_a(p) + cos(alpha) chi_b(p)`. But then the global L-function would be the product over `p` of these factors, which equals `prod_p (1 - (sin(alpha) chi_a(p) + cos(alpha) chi_b(p)) p^{-s})^{-1}`. This is NOT D-H: expanding the product gives a Dirichlet series whose coefficient at `n = p` is `sin(alpha) chi_a(p) + cos(alpha) chi_b(p)`, but whose coefficient at `n = p q` is `(sin(alpha) chi_a(p) + cos(alpha) chi_b(p))(sin(alpha) chi_a(q) + cos(alpha) chi_b(q))`, which differs from the D-H coefficient at `pq` (the latter is `sin(alpha) chi_a(p) chi_a(q) + cos(alpha) chi_b(p) chi_b(q)` because D-H = sin · L_a + cos · L_b).

**Structural obstruction at (2).** Even if one tried to "embed" D-H by taking a direct sum blueprint `B = B_a + B_b` with `B_a` realizing `L(s, chi_a)` and `B_b` realizing `L(s, chi_b)`, the zeta of the direct sum is the **product** `L(s, chi_a) * L(s, chi_b)`, not the linear combination `sin · L_a + cos · L_b`. The Lambda-blueprint category has direct sums (which give products of zetas) but no operation that produces linear combinations of zetas.

**Multiplicative monoid cardinality argument.** A Lorscheid blueprint's multiplicative monoid `B^bullet` controls the Dirichlet coefficients of the associated L-function. For D-H, the Dirichlet coefficients form an unbounded sequence (their generating function has poles on `Re(s) = 1`-ish, with the off-line zeros at `Re(s) ≈ 0.8085`). The multiplicative monoid that would generate the D-H coefficients is the set `{ sin(alpha) chi_a(n) + cos(alpha) chi_b(n) | n ∈ N }`, which is not a multiplicative monoid (the sin/cos coefficients do not satisfy a product rule under monoid operations). So no `B^bullet` exists with the right cardinality and structure.

**Verdict (xvii)**: PASS. D-H is excluded by construction in the blueprint-relation form. This matches the R2.5 prediction and the §3.3 verdict for the Borger parent.
